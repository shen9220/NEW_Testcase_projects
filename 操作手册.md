# AI 测试用例生成器 — 操作手册

## 环境要求

- **Python** 3.10+
- **Node.js** 18+
- **npm** 9+

---

## 快速启动（推荐）

配置好环境后，一条命令启动前后端：

```bash
./start.sh
```

脚本自动完成：查找可用端口 → 启动后端 → 等待就绪 → 启动前端 → 打开浏览器。运行成功后终端只显示前端地址。

> 按 `Ctrl+C` 一键关闭所有服务。

---

## 一、环境配置（仅首次）

### 1.1 后端依赖安装

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 1.2 前端依赖安装

```bash
cd frontend
npm install
```

### 1.3 配置环境变量

复制模板文件并填入实际值：

```bash
cd backend
cp .env.example .env
```

编辑 `backend/.env`，填入以下配置：

```env
# Supabase（从 https://supabase.com/dashboard 获取你的项目 URL 和 Key）
COZE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
COZE_SUPABASE_SERVICE_ROLE_KEY=eyJ...

# AI 模型 API Key
COZE_DEEPSEEK_API_KEY=sk-...
```

> 不配 `.env` 也能启动，系统会自动降级为**本地离线模式**（数据存本地 JSON 文件）。但 AI 生成功能需要 DeepSeek API Key。

---

## 二、启动后端

打开一个终端窗口：

```bash
cd backend
source venv/bin/activate
python run.py
```

> 端口机制：默认从 8000 开始寻找可用端口，若被占用自动切换至 8001、8002...（最多 20 次尝试）。终端会打印实际启动地址。

启动成功标志：

```
后端启动地址: http://localhost:8000
健康检查:     http://localhost:8000/api/v1/health

INFO: Uvicorn running on http://127.0.0.1:8000
```

验证（注意替换端口号）：

```bash
curl http://localhost:8000/api/v1/health
```

应返回：

```json
{"code":200,"data":{"backend":"ok","supabase":"connected","storage_mode":"dual"}...}
```

---

## 三、启动前端

打开另一个终端窗口（后端保持运行）：

```bash
cd frontend
npm run dev
```

> 端口机制：默认 5173，被占用时 Vite 自动尝试 5174、5175...
>
> 如果后端切换了端口，启动前端时需同步设置环境变量：
> ```bash
> VITE_BACKEND_PORT=8001 npm run dev
> ```

启动成功标志：

```
VITE v6.x.x  ready in xxx ms
➜  Local:   http://localhost:5173/
```

---

## 四、打开系统

浏览器访问终端输出的前端地址（默认 **http://localhost:5173**）

你将看到以下界面：

- 左侧：**工作台**（主操作区：上传 PRD、生成用例、查看用例表格）
- 右侧：**历史任务**（已生成的历史记录列表）
- 顶部导航：**工作台** / **日志中心** 切换
- 右上角标签：Supabase 连接状态
  - 绿色 "Supabase 已连接" — 数据双写（本地 + 云端）
  - 红色 "离线模式" — 仅本地 JSON 文件存储

---

## 五、基本操作流程

### 5.1 上传 PRD 文档

在工作台页面，选择以下任一方式输入需求：

| 方式 | 操作 |
|------|------|
| 拖拽上传 | 将 `.md` / `.docx` / `.pdf` 文件拖入上传区 |
| 手动输入 | 在 Markdown 编辑器中直接编写 PRD 内容 |

上传后会自动显示 **PRD 解析预览**。

### 5.2 生成测试用例

点击 **「生成测试用例」** 按钮，系统将运行 8 步流水线：

1. 提取功能模块
2. 逐模块生成用例
3. 提取字段约束
4. 补充边界用例
5. 状态转换补充
6. 步骤具体化
7. 语义去重
8. 后处理校验（Sanity Checker）

进度以步骤条展示，生成完成后用例表格自动刷新。

### 5.3 查看与编辑用例

- 点击某行 → 弹出**详情抽屉**（步骤明细、优先级、标签等）
- 点击编辑图标 → 弹出**编辑弹窗**（可修改所有字段）

### 5.4 导出

- 点击 **Excel** 按钮 → 下载 `.xlsx` 格式用例
- 点击 **XMind** 按钮 → 下载 `.xmind` 思维导图

### 5.5 删除用例

- 单条删除：行末删除按钮
- 批量删除：勾选多行 → 点击「批量删除」

---

## 六、关闭系统

在两台终端窗口中分别按 `Ctrl+C` 停止前端和后端。

---

## 七、常见问题

| 问题 | 解决 |
|------|------|
| 前端 5173 页面空白 | 确认后端 8000 已启动，前端 Vite 代理依赖后端 |
| 生成按钮无反应 | 检查 DeepSeek API Key 是否已配置在 `.env` 中 |
| Supabase 显示"离线模式" | 检查 `.env` 中 Supabase URL 和 Key 是否正确，以及数据库表是否已创建 |
| 端口被占用 | 已内置自动端口切换（后端 `python run.py`、前端 `strictPort: false`），无需手动处理 |
| 模块缺失报错 | 依赖未装全，重新运行 `pip install -r requirements.txt` 或 `npm install` |
