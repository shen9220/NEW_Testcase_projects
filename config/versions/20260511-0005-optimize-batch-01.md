# optimize01 批次优化 — 7项需求迭代

## Description

基于 `config/optimize/optimize01.md` 实施全部 7 项优化需求。

### 1. 操作日志完善
- 后端：新增 `middleware("http")` 记录每条 HTTP 请求（method, path, status, duration）
- 后端：新增 `POST /api/v1/logs/action` 接收前端用户操作日志
- 前端：`logFrontendAction()` fire-and-forget 记录所有点击（upload, generate, delete, export, view, edit, regenerate）

### 2. 按 PRD 模块分组展示 + 按模块下载
- `TestCaseTable` 改用 Ant Design `Collapse` 按模块分组，每组可展开查看子表
- 每个模块面板含"下载此模块"按钮，调用 Excel/XMind 导出（带 module 过滤参数）
- 后端 `export` 接口新增 `module` query param，导出服务支持按模块过滤

### 3. 左右分栏布局
- `Workbench` 改用 `Row/Col` 左右各占 50%（lg=12）
- 左侧：PRD 输入 + 解析预览 + 生成进度
- 右侧：覆盖率统计 + 测试用例表格

### 4. 模块覆盖率统计
- 右侧面板顶部展示覆盖率 `Progress` 进度条
- 显示 "已覆盖 X / Y 个模块" + 百分比
- 覆盖率 100% 显示绿色"全覆盖"标签，否则橙色"有遗漏"

### 5. 补全未覆盖模块
- 后端：新增 `POST /api/v1/generation/{task_id}/regenerate-modules`，针对未覆盖模块重新生成
- 前端：未覆盖模块旁显示"补全未覆盖模块"按钮，触发后台生成并轮询状态

### 6. 用例覆盖标注
- `TestCase` 类型新增 `prd_coverage` 字段："已覆盖"/"未覆盖"
- 表格新增"覆盖状态"列，绿色/橙色标签区分

### 7. 分页数量控制
- `TestCaseTable` 新增每页条数选择器（10/20/50 条）
- 默认每页 10 条
