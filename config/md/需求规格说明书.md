# 需求规格说明书

## 1. 概述
构建一个本地运行的 AI 测试用例生成器，帮助测试人员根据 PRD 文档快速产出高质量、可执行的测试用例。系统采用前后端分离架构，数据存储使用本地文件 + Supabase 双重持久化。

## 2. 功能需求

### 2.1 需求输入模块
| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | PRD 文件上传 | 支持 .md、.docx、.pdf 格式，拖拽或点击上传，上传后自动解析为 Markdown 预览 |
| F2 | 手动输入框 | 提供 Markdown 编辑器，可直接编写或粘贴 PRD 内容 |
| F3 | 提交按钮 | 点击后解析内容，调用 AI 按 Skill 流水线生成测试用例，显示进度（模块提取→逐模块生成→边界补充→步骤细化→去重） |
| F4 | 历史记录 | 每次提交形成一条记录，保留原始 PRD、生成结果、操作日志；支持切换查看任意历史任务 |
| F5 | Supabase 同步 | 每次操作自动同步到 Supabase，前端启动时检测连接状态并展示 |

### 2.2 测试用例管理模块
| 编号 | 功能 | 描述 |
|------|------|------|
| T1 | 用例展示 | 表格形式显示：ID、模块、标题、优先级、类型、步骤数、操作按钮 |
| T2 | 分页 | 前端分页，默认每页 20 条，可选 10/20/50 |
| T3 | 单行操作 | 查看详情（抽屉展开完整步骤）、编辑（模态框修改所有字段）、删除单条用例（逻辑删除） |
| T4 | 批量删除 | 勾选多条后批量逻辑删除，需二次确认 |
| T5 | Excel 导出 | 导出当前筛选后的用例为 .xlsx，自动下载 |
| T6 | XMind 导出 | 按「模块→用例标题」树形结构生成 .xmind，自动下载 |

### 2.3 日志模块
- 记录操作类型（上传、生成、编辑、删除、导出、批量删除）、时间、操作人（本地默认 `user`）、详细信息
- 日志列表倒序展示，支持按时间范围和操作类型过滤
- 支持全局日志和单任务日志视图

### 2.4 数据存储
- **本地文件**：存储原始上传文件、解析后 Markdown 全文、生成的用例 JSON、导出文件、全局日志
- **Supabase**：存储项目元信息、测试用例结构化数据、操作日志（支持高效查询、分页、过滤）
- 双重写入策略：每次操作同时写入本地文件和 Supabase；查询优先从 Supabase 读取，离线时回退本地 JSON
- 所有生成记录永久保留，从不覆盖

## 3. 非功能需求
| 编号 | 类型 | 描述 |
|------|------|------|
| NF1 | 性能 | 文件解析 + AI 生成 < 120 秒（30 页内文档），前端界面响应 < 500ms |
| NF2 | 可用性 | Supabase 不可用时系统仍可正常使用（自动降级为纯本地模式） |
| NF3 | 可维护性 | 代码分层清晰（router → service → repository），前端组件原子化，易于扩展新文档格式或 AI 模型 |
| NF4 | 数据安全 | .env 文件不入库（gitignore），Supabase RLS 策略保护数据，service_role_key 仅后端使用 |
| NF5 | 兼容性 | 支持 macOS / Windows / Linux，Node.js 18+，Python 3.10+ |
