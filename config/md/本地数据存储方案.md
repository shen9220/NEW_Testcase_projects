# 本地数据存储方案

## 总体策略

采用 **本地文件系统 + Supabase (PostgreSQL)** 双重存储：

- **本地文件**：存储原始文件、导出文件、以及所有数据的 JSON 副本（离线可用、不丢失）
- **Supabase**：存储结构化数据，支持高效查询、分页、过滤、搜索
- **写入策略**：每次操作同时写入两端（双写），保证数据一致性
- **读取策略**：优先从 Supabase 读取，连接失败时自动回退本地 JSON

### 数据持久化规则

- 删除操作为逻辑删除（`is_deleted: true`），数据保留在 JSON 中备查
- 前端可通过历史列表切换到任意一次生成结果

### 表结构 DDL

#### 1. projects 表（项目/任务）

```sql
CREATE TABLE projects (
    id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    task_id         VARCHAR(50) NOT NULL UNIQUE,       -- 如 20260509-143022
    original_filename VARCHAR(500) NOT NULL,            -- 原始文件名
    prd_content     TEXT,                               -- 解析后的 Markdown 全文
    status          VARCHAR(20) DEFAULT 'processing',   -- processing / completed / failed
    testcase_count  INTEGER DEFAULT 0,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_projects_task_id ON projects(task_id);
CREATE INDEX idx_projects_created_at ON projects(created_at DESC);
CREATE INDEX idx_projects_status ON projects(status);
```

#### 2. testcases 表（测试用例）

```sql
CREATE TABLE testcases (
    id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    project_id      UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    case_id         VARCHAR(20) NOT NULL,              -- 项目内编号，如 tc-001
    module          VARCHAR(200) NOT NULL,              -- 功能模块
    title           VARCHAR(500) NOT NULL,              -- 用例标题
    precondition    TEXT DEFAULT '',                     -- 前置条件
    steps           JSONB NOT NULL DEFAULT '[]',        -- [{action, expected}]
    priority        VARCHAR(5) DEFAULT 'P2',            -- P0 / P1 / P2 / P3
    type            VARCHAR(50) DEFAULT '功能测试',      -- 功能/异常/边界/性能/安全/兼容性
    tags            JSONB DEFAULT '[]',                 -- 标签数组
    notes           TEXT DEFAULT '',                     -- 备注
    is_deleted      BOOLEAN DEFAULT FALSE,              -- 逻辑删除
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),

    UNIQUE(project_id, case_id)
);

-- 索引
CREATE INDEX idx_testcases_project_id ON testcases(project_id);
CREATE INDEX idx_testcases_module ON testcases(module);
CREATE INDEX idx_testcases_priority ON testcases(priority);
CREATE INDEX idx_testcases_type ON testcases(type);
CREATE INDEX idx_testcases_is_deleted ON testcases(is_deleted);
CREATE INDEX idx_testcases_steps ON testcases USING GIN (steps);
```

#### 3. logs 表（操作日志）

```sql
CREATE TABLE logs (
    id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    project_id      UUID REFERENCES projects(id) ON DELETE SET NULL,  -- 全局日志可为空
    task_id         VARCHAR(50),                        -- 冗余字段，方便查询
    operation_type  VARCHAR(50) NOT NULL,               -- upload / generate / edit / delete / export / batch_delete
    detail          TEXT NOT NULL,                       -- 详细信息
    operator        VARCHAR(100) DEFAULT 'user',
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_logs_project_id ON logs(project_id);
CREATE INDEX idx_logs_task_id ON logs(task_id);
CREATE INDEX idx_logs_operation_type ON logs(operation_type);
CREATE INDEX idx_logs_created_at ON logs(created_at DESC);
```

#### 4. exports 表（导出记录）

```sql
CREATE TABLE exports (
    id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    project_id      UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    task_id         VARCHAR(50) NOT NULL,
    format          VARCHAR(10) NOT NULL,               -- xlsx / xmind
    file_path       VARCHAR(1000) NOT NULL,             -- 本地文件路径
    file_size       INTEGER DEFAULT 0,                  -- 文件大小（字节）
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_exports_project_id ON exports(project_id);
CREATE INDEX idx_exports_created_at ON exports(created_at DESC);
```

### RLS (Row Level Security) 策略

```sql
-- 本地单用户模式，使用 anon key 即可访问所有数据
-- 启用 RLS
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE testcases ENABLE ROW LEVEL SECURITY;
ALTER TABLE logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE exports ENABLE ROW LEVEL SECURITY;

-- 允许 anon 角色全量读取
CREATE POLICY "anon_read_all" ON projects FOR SELECT USING (true);
CREATE POLICY "anon_read_all" ON testcases FOR SELECT USING (true);
CREATE POLICY "anon_read_all" ON logs FOR SELECT USING (true);
CREATE POLICY "anon_read_all" ON exports FOR SELECT USING (true);

-- 允许 anon 角色插入
CREATE POLICY "anon_insert_all" ON projects FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_all" ON testcases FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_all" ON logs FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_all" ON exports FOR INSERT WITH CHECK (true);

-- 允许 anon 角色更新
CREATE POLICY "anon_update_all" ON projects FOR UPDATE USING (true);
CREATE POLICY "anon_update_all" ON testcases FOR UPDATE USING (true);

-- 允许 anon 角色删除（实际使用逻辑删除，此处保留以防需要物理删除）
CREATE POLICY "anon_delete_all" ON testcases FOR DELETE USING (true);
CREATE POLICY "anon_delete_all" ON logs FOR DELETE USING (true);
```

**注意**：以上 RLS 策略为本地单用户简化版。如需多用户，应增加 `user_id` 字段并按用户隔离数据。

## 三、后端数据访问层设计

### Repository 模式（双写）

```python
# repositories/project_repo.py

class ProjectRepository:
    def __init__(self, local: LocalStorage, supabase: SupabaseClient):
        self.local = local
        self.supabase = supabase

    async def create(self, project: Project) -> Project:
        # 1. 写入本地 JSON
        self.local.append_to_index(project)
        self.local.create_task_dir(project.task_id)
        # 2. 写入 Supabase
        try:
            await self.supabase.table("projects").insert(project.to_dict()).execute()
        except Exception as e:
            logger.warning(f"Supabase write failed, local-only: {e}")
        return project

    async def get_by_task_id(self, task_id: str) -> Optional[Project]:
        # 1. 优先从 Supabase 读取
        try:
            result = await self.supabase.table("projects") \
                .select("*").eq("task_id", task_id).execute()
            if result.data:
                return Project.from_dict(result.data[0])
        except Exception:
            pass
        # 2. 回退本地
        return self.local.get_from_index(task_id)

    async def list_all(self, page: int = 1, page_size: int = 20) -> PaginatedResult:
        try:
            result = await self.supabase.table("projects") \
                .select("*", count="exact") \
                .order("created_at", desc=True) \
                .range((page-1)*page_size, page*page_size-1) \
                .execute()
            return PaginatedResult(items=result.data, total=result.count)
        except Exception:
            return self.local.list_projects_paginated(page, page_size)
```

### Supabase 离线降级

```
每次操作：
  1. 本地文件写入（同步，确保不丢失）
  2. Supabase 写入（异步，失败仅记录日志）

每次查询：
  1. 尝试 Supabase
  2. 失败 → 回退本地 JSON
  3. 前端展示数据来源标记
```

## 四、日志记录规范

### 全局日志

记录于 `data/app.log`，通过 Supabase `logs` 表查询（project_id 为 NULL）。

### 任务日志

记录于 `data/{task_id}/log.txt`，同时写入 Supabase `logs` 表（带 project_id）。

### 日志级别

| 级别 | 使用场景 |
|------|----------|
| INFO | 正常操作：上传、生成开始/完成、导出、编辑 |
| WARNING | 非关键错误：Supabase 写入失败（已回退本地）、解析部分失败 |
| ERROR | 关键错误：AI 调用失败、文件写入失败 |
