# 系统架构设计

## 整体架构
前后端分离，本地统一启动。数据采用双重存储：本地文件 + Supabase。

```
┌──────────────────────────────────────────────────────────────────┐
│                      浏览器前端 (React + Vite)                     │
│                    http://localhost:5173                          │
└──────────────────────────┬───────────────────────────────────────┘
                           │ REST API (/api/v1/*)
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│                   本地后端 (FastAPI, port 8000)                    │
│  ┌──────────┐ ┌──────────────┐ ┌────────────┐ ┌──────────┐ ┌──────────┐  │
│  │ 文档解析  │ │  AI 生成引擎  │ │ Post       │ │  导出服务 │ │ 日志服务 │  │
│  │ parser   │ │ 7+1 Skill   │ │ Validator  │ │  export  │ │  logger  │  │
│  │          │ │ 流水线       │ │ (校验器)    │ │          │ │          │  │
│  └──────────┘ └──────┬───────┘ └─────┬──────┘ └──────────┘ └──────────┘  │
│                      │                                            │
│         ┌────────────┼────────────┐                               │
│         ▼            ▼            ▼                               │
│  ┌──────────┐ ┌────────────┐ ┌──────────┐                        │
│  │ 本地存储  │ │  Supabase  │ │ LLM API  │                        │
│  │ data/    │ │ PostgreSQL │ │ DeepSeek │                        │
│  └──────────┘ └────────────┘ └──────────┘                        │
└──────────────────────────────────────────────────────────────────┘
```

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + Vite + Ant Design 5 | 构建用户界面 |
| 前端导出 | xmind-v2, exceljs | 前端辅助导出 XMind/Excel |
| 后端 | Python 3.10+ FastAPI | RESTful API，异步处理 |
| AI 引擎 | DeepSeek (OpenAI 兼容接口) | 通过 `ai_engine/bot_config.yaml` 配置 |
| 文件解析 | PyMuPDF (PDF) / python-docx (Word) / mistletoe (MD) | 解析为统一 Markdown |
| Excel 导出 | openpyxl | 生成 .xlsx |
| XMind 导出 | xmind 库 | 生成标准 .xmind 文件 |
| 数据库 | Supabase (PostgreSQL) | 结构化数据持久化 |
| 本地存储 | 文件系统 (JSON + 原始文件) | 原始文件、导出文件、离线回退 |

## 双重存储策略

```
写操作：同时写入本地文件 + Supabase
读操作：优先从 Supabase 读取（支持分页/过滤/搜索）
          ↓ 失败或离线
        回退本地 JSON 文件读取
```

| 数据类型 | 本地存储 | Supabase |
|----------|----------|----------|
| 原始上传文件 (.md/.docx/.pdf) | ✅ data/{task_id}/original.* | ❌ |
| 解析后 Markdown | ✅ data/{task_id}/prd_content.md | ✅ projects.prd_content |
| 测试用例 | ✅ data/{task_id}/testcases.json | ✅ testcases 表 |
| 导出文件 (.xlsx/.xmind) | ✅ data/{task_id}/export/ | ✅ exports 表（路径引用） |
| 操作日志 | ✅ data/app.log + data/{task_id}/log.txt | ✅ logs 表 |
| 项目索引 | ✅ data/projects_index.json | ✅ projects 表 |

## 数据流（测试用例生成全流程）

```
1. 用户上传 PRD 文件
   FileUpload ──→ POST /api/v1/documents/upload
   ──→ parser.py 解析为 Markdown
   ──→ 保存原始文件到 data/{task_id}/original.*
   ──→ 保存 Markdown 到 data/{task_id}/prd_content.md
   ──→ 写入 projects 表（Supabase）
   ──→ 返回 document_id

2. 用户点击「生成测试用例」
   SubmitButton ──→ POST /api/v1/generation/start {document_id}
   ──→ ai_generator.py 启动 Skill 流水线：
       Skill 1: extract_modules    → 提取功能模块 + 中间表示
       Skill 2: generate_for_module → 逐模块生成用例（并行）
       Skill 3: extract_constraints → 提取字段约束
       Skill 4: boundary_completion → 补充边界用例
       Skill 5: state_transition   → 补充状态转换用例
       Skill 6: refine_steps       → 细化所有步骤
       Skill 7: deduplicate        → 语义去重
       Skill 8: post_validation    → 规则校验（不通过→重试，最多2次）
   ──→ 保存用例到 data/{task_id}/testcases.json
   ──→ 批量写入 testcases 表（Supabase）
   ──→ 记录日志到 log.txt + logs 表
   ──→ 返回 task_id + 用例列表

3. 前端轮询状态（或等待完成）
   GET /api/v1/generation/{task_id}/status
   ──→ 返回当前 Skill 进度
```

## 本地启动流程

1. 后端启动时检查 `data/` 目录，不存在则创建
2. 后端启动时尝试连接 Supabase，设置连接状态标志
3. 前端通过 Vite 代理 `/api` 到 `localhost:8000`
4. 前端启动时调用 `/api/v1/health` 检测后端 + Supabase 状态
5. Supabase 不可用时自动降级为纯本地模式
