# 后端接口设计

## 基础信息

- **Base URL**: `http://localhost:8000/api/v1`
- **Content-Type**: `application/json`
- **文件上传**: `multipart/form-data`
- **文件下载**: `application/octet-stream`

## 统一响应格式

```json
{
  "code": 200,
  "data": {},
  "message": "success"
}
```

| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 422 | 需求信息不足，无法生成用例 |
| 500 | 服务器内部错误 |
| 503 | Supabase 不可用（已降级） |

## 1. 系统

### 1.1 健康检查

```
GET /health
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "backend": "ok",
    "supabase": "connected",
    "storage_mode": "dual"
  }
}
```

`supabase` 可能的值：`connected` / `disconnected`
`storage_mode` 可能的值：`dual` / `local_only`

## 2. 文档上传与解析

### 2.1 上传文件

```
POST /documents/upload
Content-Type: multipart/form-data
```

**参数**：`file` (必填，支持 .md / .docx / .pdf，最大 50MB)

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "document_id": "20260509-143022",
    "task_id": "20260509-143022",
    "original_filename": "用户注册PRD.md",
    "prd_content": "# 用户注册\n\n用户输入手机号...",
    "file_size": 12345
  }
}
```

### 2.2 手动输入 PRD

```
POST /documents/raw
Content-Type: application/json
```

**请求体**：
```json
{
  "content": "# 用户注册\n\n用户输入手机号（11位中国大陆手机号）...",
  "title": "用户注册模块"
}
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "document_id": "20260509-143100",
    "task_id": "20260509-143100",
    "original_filename": "用户注册模块.md",
    "prd_content": "# 用户注册\n\n用户输入手机号..."
  }
}
```

### 2.3 获取文档内容

```
GET /documents/{task_id}/content
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "task_id": "20260509-143022",
    "original_filename": "用户注册PRD.md",
    "prd_content": "# 用户注册\n\n...",
    "created_at": "2026-05-09T14:30:22"
  }
}
```

## 3. 测试用例生成

### 3.1 开始生成

```
POST /generation/start
Content-Type: application/json
```

**请求体**：
```json
{
  "task_id": "20260509-143022"
}
```

**响应示例**（异步，立即返回）：
```json
{
  "code": 200,
  "data": {
    "task_id": "20260509-143022",
    "status": "processing",
    "message": "生成已开始，正在提取功能模块..."
  }
}
```

### 3.2 查询生成状态

```
GET /generation/{task_id}/status
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "task_id": "20260509-143022",
    "status": "processing",
    "progress": {
      "current_skill": "generate_for_module",
```

`status` 可能的值：`processing` / `completed` / `failed` / `partial`（部分模块成功，部分跳过或校验未完全通过）

## 4. 测试用例管理

### 4.1 分页获取用例

```
GET /testcases?task_id={task_id}&page=1&page_size=20&module=&priority=&type=&keyword=
```

所有查询参数可选。`keyword` 支持在标题和模块中搜索。

### 4.2 获取单条用例详情

```
GET /testcases/{case_id}
```

### 4.3 更新用例

```
PUT /testcases/{case_id}
Content-Type: application/json
```

**请求体**（所有字段可选，只传需要修改的字段）：
```json
{
  "module": "用户注册",
  "title": "修改后的标题",
  "precondition": "修改后的前置条件",
  "steps": [
    {"action": "更新后的操作", "expected": "更新后的预期"}
  ],
  "priority": "P1",
  "type": "异常测试",
  "tags": ["regression"],
  "notes": "修改备注"
}
```

### 4.4 删除单条用例（逻辑删除）

```
DELETE /testcases/{case_id}
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "case_id": "tc-001",
    "is_deleted": true,
    "message": "用例已删除（逻辑删除，数据保留备查）"
  }
}
```

### 4.5 批量删除

```
POST /testcases/batch-delete
Content-Type: application/json
```

**请求体**：
```json
{
  "case_ids": ["tc-001", "tc-003", "tc-005"]
}
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "deleted_count": 3,
    "deleted_ids": ["tc-001", "tc-003", "tc-005"]
  }
}
```

```
GET /export/excel?task_id={task_id}
```

```
GET /export/xmind?task_id={task_id}
```

## 数据模型（Pydantic / TypeScript）

### TestCase（用例）
```python
class TestStep(BaseModel):
    action: str          # 具体操作步骤
    expected: str        # 可量化预期结果

class TestCase(BaseModel):
    id: str              # UUID，数据库主键
    case_id: str         # 项目内编号，如 tc-001
    module: str          # 功能模块
    title: str           # 用例标题
    precondition: str    # 前置条件（默认 ""）
    steps: list[TestStep] # 步骤列表
    priority: str        # P0 / P1 / P2 / P3
    type: str            # 功能/异常/边界/性能/安全/兼容性
    tags: list[str]      # 标签
    notes: str           # 备注
    is_deleted: bool     # 逻辑删除标记
    created_at: str      # ISO 时间戳
    updated_at: str      # ISO 时间戳
```

### Project（项目/任务）
```python
class Project(BaseModel):
    id: str              # UUID
    task_id: str         # YYYYMMDD-HHMMSS 格式
    original_filename: str
    prd_content: str     # Markdown 全文
    status: str          # processing / completed / failed
    testcase_count: int
    created_at: str
    updated_at: str
```
