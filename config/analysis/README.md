# 问题分析与解决方案

## 目录规范

- `analysis-NN-简短描述.md` — 每次问题的根因分析与解决方案
- NN 为两位数字序号，如 `analysis-01-xxx.md`

## 工作流程

1. **用户提出 bug/问题** → Claude 深入代码追踪分析
2. **生成 analysis 文档** → 列出所有失败点 + 解决方案
3. **用户审查确认** → 通过后按方案执行实施
4. **实施完成后** → 在 `config/versions/` 记录版本迭代文档
